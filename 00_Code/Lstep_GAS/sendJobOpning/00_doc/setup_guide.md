## 1. スプレッドシートの準備

Google スプレッドシートを新規作成し、以下のシートをすべて作成します。
**シート名は以下の通り正確に入力してください（大文字・小文字・スペースを含む）。**

| シート名 | 用途 |
| --- | --- |
| `シート1` | LStepフォーム回答データ（LStepが自動書き込み） |
| `Job_Opening` | 求人情報マスタ |
| `ID_MAP` | ユーザーごとの送信済み求人履歴 |
| `Form_Group` | フォーム回答と求人グループのマッピング |
| `Log` | 業務処理ログ（送信成功/失敗） |
| `Answer_ID_Log` | 回答ID単位の処理ログ（重複送信防止用） |
| `Logger` | システム内部ログ |
| `Setting` | システム設定値 |

### 1.1 各シートのヘッダー行を設定する

各シートの **1行目** に以下のヘッダーを入力してください。

#### `Job_Opening` シート（A1〜G1）

| A | B | C | D | E | F | G |
| --- | --- | --- | --- | --- | --- | --- |
| ID | Vaild | Group_ID | Url | Job_Name | Salary | Working_Style |

> **注意：** `Vaild` は Valid のスペルミスですが、コードがこの名前で参照しているため変更しないでください。

B列（Vaild）はチェックボックス形式にします。
**B列全体を選択 → 挿入 → チェックボックス** で設定してください。

> **[画像が必要] Job_Openingシートのヘッダー行とB列チェックボックス設定の画面**

---

#### `ID_MAP` シート（A1〜B1）

| A | B |
| --- | --- |
| USER_ID | Send_Letter_ID |

#### `Form_Group` シート（A1のみ先に設定）

> **注意：** このシートのヘッダー列名はLStepフォームの質問文と完全一致させる必要があります。
> **[ステップ6でFORM_HEADERをカスタマイズした後に、このシートのヘッダーを設定してください。]**

ここでは A1 に `Group_ID` だけ入力しておきます。B1以降は **ステップ6完了後** に入力します。

| A |
| --- |

#### `Log` シート（A1〜F1）

| A | B | C | D | E | F |
| --- | --- | --- | --- | --- | --- |
| Timestamp | Status | 回答ID | 回答日時 | 回答者ID | 回答者名 |

---

#### `Answer_ID_Log` シート（A1〜F1）

| A | B | C | D | E | F |
| --- | --- | --- | --- | --- | --- |
| Timestamp | Status | 回答ID | 回答日時 | 回答者ID | 回答者名 |

## 2. GASプロジェクトの作成とコードの設置

### 2.1 スクリプトエディタを開く

スプレッドシートのメニューから **拡張機能 → Apps Script** を開きます。

> **[画像が必要] スプレッドシートの「拡張機能」→「Apps Script」メニューの画面**

### 2.2 ファイルを作成してコードをコピーする

Apps Script エディタで以下のファイルを作成し、それぞれ対応するコードを貼り付けます。

| 作成するファイル名 | コピー元ファイル |
| --- | --- |
| `returnLINE` | `100_Code/returnLINE.gas` |
| `Mapping` | `100_Code/Mapping.gas` |
| `AddJobData` | `100_Code/AddJobData.gas` |
| `JobMange` | `100_Code/JobMange.gas` |
| `fetchURL` | `100_Code/fetchURL.gas` |

また、HTMLファイルも追加します。

| 作成するHTMLファイル名 | コピー元ファイル |
| --- | --- |
| `JobModal` | `100_Code/JobModal.html` |
| `JobManageModal` | `100_Code/JobManageModal.html` |

> **[画像が必要] Apps Scriptでファイルを追加する操作画面（「＋」ボタンからスクリプトやHTMLを追加する画面）**

### 2.3 onOpen の重複を解消する

> **重要：** `AddJobData.gas` と `JobMange.gas` の両方に `onOpen` 関数が定義されています。
> GASでは同名関数が複数あると片方が無視され、メニューが正常に表示されません。
> **必ずどちらか一方の `onOpen` を削除し、もう一方に統合してください。**

統合後の `onOpen`（`JobMange.gas` 側に残す場合の例）：

```javascript
function onOpen() {
  SpreadsheetApp.getUi()
    .createMenu("求人管理")
    .addItem("求人を登録", "openJobModal")
    .addItem("有効/無効を一括設定", "openJobManageModal")
    .addToUi();
}
```

`AddJobData.gas` 側の `onOpen` 関数（1〜6行目）は**削除**してください。

### 2.4 プロジェクトを保存する

Ctrl+S（Mac: Cmd+S）で保存します。エラーが出ていないことを確認してください。

3. **「スクリプト プロパティを追加」** をクリック
4. 以下の値を入力して保存

| --- | --- |
| `LSTEP_API_TOKEN` | LStepの管理画面から取得したAPIトークン |

> **[画像が必要] Apps Scriptのプロジェクト設定画面でスクリプトプロパティを追加する画面**

### LStep APIトークンの取得方法

## 4. LStep APIエンドポイントURLの書き換え

**LStepの自分のアカウントのURLに書き換えてください。**

```javascript
function sendLstepAPI(friend_id, message) {
  const url = "https://api.lineml.jp/v1/api-codes/【アカウントID】/triggers/【トリガーID】";
  // ↑ この URL を自分のLStepアカウントのものに書き換える
```

### LStep APIエンドポイントURLの確認方法

LStepの管理画面から外部連携 → API設定 → トリガーURLを確認してください。

## 6. FORM_HEADERのカスタマイズ（コード変更）

`returnLINE.gas` の冒頭にある `FORM_HEADER` 定数を、**実際のLStepフォームの質問文に合わせて**書き換えます。
**この設定が後続のForm_Groupシート設定と直接連動するため、必ずここで先に確定させてください。**

### デフォルト設定（`returnLINE.gas` 18〜24行目）

```javascript
const FORM_HEADER = {
  jobType: "希望する職種",           // ← LStepフォームの質問文に合わせて変更
  area:    "希望する勤務エリア",      // ← 同上
  age:     "年代を教えてください。",  // ← 同上
  skill:   "宅建を持っていますか？",  // ← 同上
  selfPR:  "自己PRを教えてください（任意）",
  other:   "その他の回答"
};
```

### 変更ルール

- **左側（`jobType:` など）**：変更不要（コード内部で使うキー名）
- **右側（`"希望する職種"` など）**：LStepフォームの質問文と**完全一致**させる（句読点・スペース含む）
- 不要な質問項目は行ごと削除してください
- 質問が増えた場合は行を追加してください

### 注意点

`FORM_HEADER` の値（右辺の文字列）は以下の3か所すべてで完全一致している必要があります：

| 場所 | 該当箇所 |
| --- | --- |
| `FORM_HEADER`（コード） | 右辺の文字列 |
| `シート1` | 1行目のヘッダー列名 |
| `Form_Group` シート | 1行目のヘッダー列名（B列以降） |

## 7. Form_Groupシートの設定（回答マッピング）

このシートが、フォームの回答内容と「どの求人を送るか」を紐付ける最も重要な設定です。
**ステップ6でFORM_HEADERを確定した後に行ってください。**

### 7.1 Form_Groupシートのヘッダーを設定する

ステップ1で `Group_ID` だけ入力したヘッダー行に、B列以降を追加します。
**B1以降はFORM_HEADERの値（右辺の文字列）と完全一致する列名を入力してください。**

例（デフォルトFORM_HEADERの場合）：

| A | B | C | D | E |
| --- | --- | --- | --- | --- |
| Group_ID | 希望する職種 | 希望する勤務エリア | 年代を教えてください。 | 宅建を持っていますか？ |

### 7.2 回答パターンとGroup_IDのデータを入力する

2行目以降に、回答の組み合わせとGroup_IDを入力します。

#### 設定の考え方

- **Group_ID**：求人グループを識別するID（数字推奨、例：1, 2, 3…）
- **回答列の値**：該当する回答を入力。**空欄はワイルドカード**（すべての回答に一致）

#### 設定例

以下のようなLStepフォームがある場合：
- 質問1：希望する職種（営業 / 事務 / IT）
- 質問2：希望する勤務エリア（東京 / 大阪 / 名古屋）

| Group_ID | 希望する職種 | 希望する勤務エリア | 年代を教えてください。 | 宅建を持っていますか？ |
| --- | --- | --- | --- | --- |
| 1 | 営業 | 東京 | | |
| 2 | 営業 | 大阪 | | |
| 3 | 事務 | | | |
| 4 | | | | はい |

上記の場合：
- 「営業・東京」と回答したユーザーには Group_ID=1 の求人を送信
- 「事務・名古屋」と回答したユーザーには Group_ID=3 の求人を送信（エリア列が空欄＝ワイルドカード）
- 複数のGroup_IDに一致した場合は、それぞれのグループの求人を合算して送信

> **[画像が必要] Form_Groupシートのヘッダー・データ入力例の画面**

### 7.3 Job_OpeningシートのGroup_IDと対応させる

Form_Groupに登録したGroup_IDと同じ値を、Job_Openingの `Group_ID` 列に使用します。
求人登録モーダル（ステップ10.1）でGroup_IDを入力する際に参照してください。

## 8. LStepフォームのシート連携設定

| 2列目 | 回答日時 |
| 3列目 | 回答者ID（LStepユーザーID） |
| 4列目 | 回答者名 |
| 5列目以降 | 各質問の回答（FORM_HEADERに定義した順に合わせる） |

## 9. トリガーの設定

GASのトリガーを2つ設定します。

### 9.1 onSheetChange（メイン処理トリガー）

フォーム回答がシート1に書き込まれたときに自動実行されるトリガーです。

1. Apps Script エディタの左メニューから **トリガー**（時計アイコン）を開く
2. 右下の **「トリガーを追加」** をクリック
3. 以下の設定で作成

| 設定項目 | 値 |
| --- | --- |
| 実行する関数を選択 | `onSheetChange` |
| 実行するデプロイを選択 | Head |
| イベントのソースを選択 | スプレッドシートから |
| イベントの種類を選択 | 変更時 |

> **[画像が必要] GASトリガー設定画面（onSheetChange）**

### 9.2 deleteOldLogs（ログ自動削除トリガー）

Loggerシートのログが肥大化しないよう、7日以上前のログを定期削除するトリガーです。

1. 同じく **「トリガーを追加」** をクリック
2. 以下の設定で作成

| 設定項目 | 値 |
| --- | --- |
| 実行する関数を選択 | `deleteOldLogs` |
| 実行するデプロイを選択 | Head |
| イベントのソースを選択 | 時間主導型 |
| 時間ベースのトリガーのタイプ | 日付ベースのタイマー |
| 時刻を選択 | 午前1時〜2時（任意） |

> **[画像が必要] GASトリガー設定画面（deleteOldLogs）**

## 10. 動作確認

### 10.1 求人を1件登録してみる

スプレッドシートのメニューに **「求人管理」** が追加されているはずです。
（表示されない場合はページを再読み込みしてください）

1. **求人管理 → 求人を登録** をクリック
2. モーダルが開くことを確認
3. Group_IDとHOMESの求人URLを入力して登録
4. `Job_Opening` シートに行が追加されていることを確認

> **[画像が必要] 「求人管理」メニューと求人登録モーダルの画面**

### 10.2 フォーム回答のシミュレーション

`シート1` に手動でテストデータを1行入力し、以下を確認します。

- `Log` シートにログが記録されているか
- `Answer_ID_Log` シートにログが記録されているか
- LStepユーザーへメッセージが送信されているか

### 10.3 エラー時の確認

エラーが発生した場合は `Logger` シートを確認してください。
`Level` 列が `ERROR` の行に詳細が記録されています。

# 求人自動送信システム ユーザー向け操作マニュアル

---

## 1. システム概要

LINEのフォーム（LStep）に回答したユーザーへ、回答内容に合った求人情報を自動でLINEに送信するシステムです。

Google スプレッドシートをデータベースとして使用しており、求人情報の登録・管理はスプレッドシートのメニューから行います。

---

## 2. システムの全体イメージ

```
ユーザー（LINE）
    │
    ↓ フォーム回答
LStep（LINE拡張ツール）
    │
    ↓ 回答データ転送
Google スプレッドシート（シート1）
    │
    ↓ 自動トリガー起動
GAS（自動処理）
    │
    ├─ 回答内容 → 求人グループ照合
    ├─ 未送信求人を取得
    └─ LINEへ求人情報を自動送信
```

---

## 3. 事前準備（初回のみ）

### 3.1 スプレッドシートの構成

以下のシートが存在していることを確認してください。

| シート名      | 用途                               |
| ------------- | ---------------------------------- |
| `Job_Opening` | 求人情報マスタ                     |
| `ID_MAP`      | ユーザーと送信済み求人の記録       |
| `Form_Group`  | フォーム回答と求人グループの対応表 |
| `シート1`     | LStepフォームの回答データ          |
| `Log`         | 送信結果ログ                       |
| `Logger`      | システム動作ログ                   |

### 3.2 Form_Group シートの設定

フォームの回答パターンに対応する「グループID」を定義します。

| Group_ID | Answer1 | Answer2 | Answer3 |
| -------- | ------- | ------- | ------- |
| 1        | a       | b       | c       |
| 2        | d       | e       | f       |

- フォームの設問と回答値に合わせて記入してください
- グループIDは求人登録時に使用します

### 3.3 スクリプトプロパティの設定

GASエディタで以下のプロパティを設定してください。

| キー              | 値                              |
| ----------------- | ------------------------------- |
| `LSTEP_API_TOKEN` | LStep から発行されたAPIトークン |

> **設定手順：** GASエディタ → 「プロジェクトの設定」 → 「スクリプトプロパティ」 → プロパティを追加

### 3.4 トリガーの設定

`onSheetChange` 関数をスプレッドシートの変更イベントに設定してください。

> **設定手順：** GASエディタ → 「トリガー」 → 「トリガーを追加」
>
> - 実行する関数：`onSheetChange`
> - イベントのソース：スプレッドシートから
> - イベントの種類：変更時

## 4. 求人の登録方法

### 4.1 メニューを開く

スプレッドシートを開き、上部メニューから **「求人管理」→「求人を登録」** を選択します。

### 4.2 登録フォームへの入力

| 項目       | 説明                                           | 例                            |
| ---------- | ---------------------------------------------- | ----------------------------- |
| Group_ID   | 送信対象のグループID（Form_Groupで定義したID） | `1`                           |
| URL        | 求人サイトのURL                                | `https://example.com/job/123` |
| 有効にする | チェックを入れると送信対象になる               | ✓チェック                     |

### 4.3 登録の確認と実行

1. 「登録内容を確認」ボタンをクリック
2. 確認欄に入力内容が表示されます
   - **Job名・給与・勤務形態はURLから自動取得されます**
3. 内容を確認後、「登録する」ボタンをクリック
4. 完了ダイアログが表示されたら登録完了です

> **注意：** URLからの情報取得に数秒かかる場合があります。

---

## 5. 求人の有効・無効の管理

### 5.1 管理画面を開く

**「求人管理」→「有効/無効を一括設定」** を選択します。

### 5.2 一覧の確認と検索

- 登録されているすべての求人が一覧表示されます
- 検索ボックスに **Group_ID・URL・Job名** のいずれかを入力して絞り込めます

### 5.3 有効・無効の切り替え

- 各行の「Vaild」列のチェックボックスで有効/無効を切り替えます
  - ✓チェックあり = **有効**（送信対象）
  - チェックなし = **無効**（送信対象外）

### 5.4 保存

「一括保存」ボタンをクリックすると変更が反映されます。

## 6. ログの確認方法

### 6.1 送信結果ログ（Log シート）

LINE送信の成功・失敗を確認できます。

| 列        | 内容                    |
| --------- | ----------------------- |
| Timestamp | 処理日時                |
| Status    | `送信成功` / `送信失敗` |
| 回答ID    | フォーム回答ID          |
| 回答日時  | フォーム回答日時        |
| 回答者ID  | LStepユーザーID         |
| 回答者名  | 回答者名                |

### 6.2 システムログ（Logger シート）

システムの詳細な動作ログを確認できます。エラー調査時に使用します。

| 列        | 内容                      |
| --------- | ------------------------- |
| Timestamp | 処理日時                  |
| Level     | `INFO` / `WARN` / `ERROR` |
| Function  | 実行された関数名          |
| Step      | 処理ステップ              |
| Message   | メッセージ                |
| Detail    | 詳細情報                  |

> **注意：** Loggerシートのログは **7日間** で自動削除されます。

## 7. エラー時の対応

### 送信が失敗した場合

1. **Log シート** で `送信失敗` の行を確認
2. **Logger シート** で同時刻の `ERROR` レベルのログを確認
3. よくある原因：
   - 回答者ID（LStepのユーザーID）が空
   - 送信可能な未送信求人がない（同グループの求人をすべて送信済み）
   - LStep APIトークンの期限切れ・設定ミス

### LINE名重複エラー

システムが回答者を特定できない場合、設定した **LINEグループ** にエラー通知が送られます。

通知を受け取った場合は手動で対応してください。

> **注意：** エラー通知先のLINEグループIDは `Setting` シートの `LINE_ERROR_GROUP_ID` に設定が必要です。

---

## 8. 注意事項

- 求人の `Vaild`（有効フラグ）が **無効** の求人はユーザーへ送信されません
- 一度送信した求人は同一ユーザーに再送信されません（`ID_MAP` シートで管理）
- 求人URLは **HOMES（homes.co.jp）** の求人ページに対応しています
- 1回の送信で届く求人数は **最大2件** に設定されています

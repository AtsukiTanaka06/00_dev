# 求人自動送信システム ユーザーマニュアル

---

## このマニュアルについて

このシステムは、**LINEユーザーがフォームに回答すると、その回答内容に合った求人情報を自動でLINEに送信する**仕組みです。

普段の業務で行う操作は主に以下の2つです。

1. **求人を登録する**（新しい求人を追加する）
2. **求人の有効・無効を管理する**（送信対象を切り替える）

ログの確認方法も本マニュアルに記載しています。

## 1. システムの全体像

### 何が自動で行われるか

```
LINEユーザーがフォームに回答する
        ↓
LStepがスプレッドシートに回答データを書き込む
        ↓
システムが自動で起動する
        ↓
回答内容に合った求人グループを探す
        ↓
まだ送っていない求人を選ぶ（最大2件）
        ↓
ユーザーのLINEに求人情報を自動送信する
```

### 送信されるメッセージの例

```
1
■求人名：○○不動産　営業スタッフ
■給与：月給25万円〜
■勤務形態：週5日 9:00〜18:00
詳しくはこちら↓
https://www.homes.co.jp/...
```

## 3. 求人を登録する

新しい求人をシステムに追加する手順です。

### 3-1. 求人登録画面を開く

スプレッドシートのメニューから **「求人管理」→「求人を登録」** をクリックします。

> **[画像] 「求人管理」メニューを開いて「求人を登録」をクリックしているところ**

数秒後に求人登録ダイアログが表示されます。

> **[画像] 求人登録モーダルダイアログの全体画面**

### 3-2. 入力項目を埋める

| 項目       | 入力内容                                         | 例                                                    |
| ---------- | ------------------------------------------------ | ----------------------------------------------------- |
| グループID | この求人を送る対象グループの番号                 | `1`                                                   |
| 求人URL    | HOMESの求人ページURL                             | `https://www.homes.co.jp/fudousantensyoku/job/xxxxx/` |
| 有効にする | 登録直後から送信対象にする場合はチェックを入れる | ✓                                                     |

> **[画像] 各入力欄に値を入れた状態のモーダル画面**

**グループIDについて：**
グループIDは、どの回答パターンのユーザーに送るかを決める番号です。
どのグループIDを使うかは `Form_Group` シートを参照してください。

> **[画像] Form_Group シートの一覧（Group_ID列と各回答条件列が並んでいる状態）**

### 3-3. 登録ボタンを押す

入力が終わったら「登録」ボタンをクリックします。

- 求人名・給与・勤務形態はURLのページから**自動で取得**されます（入力不要です）。
- 登録が完了すると成功メッセージが表示されます。

> **[画像] 登録成功後のダイアログ（「登録しました」などのメッセージ表示）**

### 3-4. 登録結果を確認する

`Job_Opening` シートを開いて、登録した求人が末尾に追加されていることを確認してください。

> **[画像] Job_Opening シートに新しい行が追加されている状態**

| 列名          | 内容                                         |
| ------------- | -------------------------------------------- |
| ID            | 自動で付与される番号                         |
| Vaild         | 有効フラグ（チェックが入っていると送信対象） |
| Group_ID      | 登録時に入力したグループID                   |
| Url           | 入力した求人URL                              |
| Job_Name      | 自動取得された求人名                         |
| Salary        | 自動取得された給与情報                       |
| Working_Style | 自動取得された勤務形態                       |

## 4. 求人の有効・無効を管理する

登録済みの求人を送信対象にするか外すかを一括で管理できます。

### 4-1. 求人管理画面を開く

スプレッドシートのメニューから **「求人管理」→「有効/無効を一括設定」** をクリックします。

> **[画像] 「求人管理」メニューを開いて「有効/無効を一括設定」をクリックしているところ**

### 4-2. 一覧画面で確認する

登録されている全求人が一覧表示されます。

> **[画像] 求人管理モーダルの一覧画面（ID・求人名・有効チェックボックスが並んでいる状態）**

| 表示項目   | 内容                               |
| ---------- | ---------------------------------- |
| ID         | 求人の番号                         |
| 求人名     | 登録時に自動取得された求人名       |
| グループID | 送信対象のグループ番号             |
| 有効       | チェックが入っている求人が送信対象 |

### 4-3. 有効・無効を切り替える

送信したくない求人のチェックを**外す**、送信したい求人に**チェックを入れる**、と操作してから「保存」ボタンをクリックします。

> **[画像] チェックボックスをオン・オフ操作している状態のモーダル画面**

> **[画像] 「保存」ボタンをクリックした後の成功メッセージ**

### 4-4. シートで直接確認する場合

`Job_Opening` シートの **`Vaild`** 列でも確認・変更できます。

> **[画像] Job_Opening シートのVaild列（チェックボックスが並んでいる状態）**

- チェックあり（TRUE）→ 送信対象
- チェックなし（FALSE）→ 送信対象外

## 5. ログを確認する

送信が正常に行われたかどうかは、以下のシートで確認できます。

### 5-1. Log シート（業務ログ）

フォームへの回答1件ごとの送信結果が記録されます。

> **[画像] Log シートの一覧（Timestamp・Status・回答ID などの列が並んでいる状態）**

| 列名      | 内容                             |
| --------- | -------------------------------- |
| Timestamp | 処理が行われた日時               |
| Status    | `送信成功` または `送信失敗`     |
| 回答ID    | フォームの回答ID                 |
| 回答日時  | ユーザーがフォームに回答した日時 |
| 回答者ID  | LStepのユーザーID                |
| 回答者名  | 回答したユーザーの名前           |

**ステータスの見方：**

| Status     | 意味                                 |
| ---------- | ------------------------------------ |
| `送信成功` | 問題なくLINEに求人を送れた           |
| `送信失敗` | 何らかのエラーが発生して送れなかった |

### 5-2. Answer_ID_Log シート（重複防止ログ）

同じ回答が2回処理されないよう、回答IDが記録されます。
通常は確認不要ですが、「同じ人に重複して送られた」「送られなかった」と報告があったときに参照してください。

> **[画像] Answer_ID_Log シートの一覧画面**

### 5-3. Logger シート（システムログ）

システム内部の詳細なログが記録されます。エラーが発生したときの原因調査に使います。

> **[画像] Logger シートの一覧（Level・Function・Step・Message 列が並んでいる状態）**

| Level   | 意味             |
| ------- | ---------------- |
| `INFO`  | 正常な処理の記録 |
| `WARN`  | 注意が必要な状態 |
| `ERROR` | エラーが発生した |

`ERROR` の行があった場合は管理者に連絡してください。

> **注意：** Logger シートのログは7日で自動削除されます。記録を残したい場合は別途コピーしてください。

## 6. よくあるトラブルと対処法

### Q. フォームに回答したのにLINEに求人が届かない

**確認手順：**

1. `Log` シートを開き、該当の回答IDの行を探す
2. `Status` 列が `送信失敗` になっていないか確認する
3. `Logger` シートで `ERROR` の行がないか確認する

**よくある原因：**

| 原因                                               | 確認場所                           | 対処                                           |
| -------------------------------------------------- | ---------------------------------- | ---------------------------------------------- |
| 回答者IDが空だった（フォームにIDが含まれていない） | Log シートの「回答者ID」列         | LStep側のフォーム設定を確認する                |
| 送信できる求人がなかった                           | Log シートの Status                | Job_Opening シートに有効な求人があるか確認する |
| 同じ回答が重複して送信されてスキップされた         | Answer_ID_Log シートで回答IDを検索 | 通常は問題なし（重複防止の正常動作）           |

---

### Q. 「求人管理」メニューが表示されない

スプレッドシートを**再読み込み（F5キーまたはブラウザの更新ボタン）**してください。

> **[画像] ブラウザの更新ボタンの位置（アドレスバー左の矢印アイコン）**

それでも表示されない場合は管理者に連絡してください。

---

### Q. 求人を登録したが「Job_Name」「Salary」などが空欄になっている

求人URLからの情報取得（スクレイピング）が失敗した可能性があります。

**確認事項：**

- URLが正しいHOMESの求人ページURLになっているか
- URLに直接アクセスして求人ページが表示されるか

> **対応サイト：** 現在はHOMES（`homes.co.jp`）の求人ページのみ自動取得に対応しています。他のサイトのURLを登録した場合、情報の自動取得はできません（URLのみ登録されます）。

---

### Q. 同じユーザーに同じ求人が2回送られた

通常、一度送った求人は再送しない仕組みになっています。
`ID_MAP` シートで該当ユーザーの送信済みIDリストを確認してください。

> **[画像] ID_MAP シートの一覧（USER_ID列と Send_Letter_ID列が並んでいる状態）**

| 列名           | 内容                                    |
| -------------- | --------------------------------------- |
| USER_ID        | LStepのユーザーID                       |
| Send_Letter_ID | 送信済み求人IDのリスト（例：`[1,2,3]`） |

---

### Q. エラー通知がLINEグループに届かない

`Setting` シートの `LINE_ERROR_GROUP_ID` が正しく設定されているか確認してください。

> **[画像] Setting シートの一覧（A列にキー、B列に値が入っている状態）**

| A列（キー）           | B列（値）        |
| --------------------- | ---------------- |
| `LINE_ERROR_GROUP_ID` | LINEグループのID |

## 付録：シート一覧

| シート名        | 役割                             | 通常の操作                     |
| --------------- | -------------------------------- | ------------------------------ |
| `シート1`       | LStepのフォーム回答データ        | 自動書き込み（触らない）       |
| `Job_Opening`   | 求人情報マスタ                   | 求人登録・有効無効の管理       |
| `Form_Group`    | 回答パターンとグループIDの対応表 | 初期設定時のみ（通常触らない） |
| `ID_MAP`        | ユーザーごとの送信済み求人ID     | 自動更新（触らない）           |
| `Log`           | 業務処理ログ                     | 確認のみ                       |
| `Answer_ID_Log` | 回答ID単位の処理ログ             | 確認のみ                       |
| `Logger`        | システム内部ログ                 | 確認のみ（7日で自動削除）      |
| `Setting`       | システム設定値                   | 初期設定時のみ                 |

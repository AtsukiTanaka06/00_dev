### 開発区分ごとの整理

**✅ Notion設定のみ（今すぐ着手可能）**

- ④ 案件進行状況のNotion管理
- Notion進捗ステータス自動更新

**🔧 ノーコード〜ローコードツールで対応（Make / Zapier / GAS）**

- ③ 契約書作成の自動化（一部）
- Googleドライブ顧客フォルダ＋ファイル複製
- 議事録送付リマインダー
- 納品後フォローアップ連絡
- 毎週月曜の挨拶連絡
- 請求書作成（Misoca）

**🛠️ カスタム開発が必要（API連携スクリプト or アプリ構築）**

- ① 議事録〜リサーチのAI実施
- ② プレゼン資料の自動作成
- ⑤ 配信動線から配信内容ベースの自動生成
- LINEノート定型文投稿（LINE Messaging APIの仕様上、ノート投稿は非対応のため代替手段の検討が必要）
- 月次レポート作成

## ① 議事録〜リサーチのAI実施

**対象フェーズ**：フェーズ4（概要設計開始〜MTG準備）

**現状の課題：**

- キックオフMTGの内容を議事録に整理するのに時間がかかる
- STP・4P・3C・BMCなどの分析を毎回AIに手動で入力して生成している

**自動化アイデア：**

- MTGの録音・録画テキスト（Zoom文字起こしなど）を入力として受け取る
- AIが自動で議事録を整理し、Notionへ格納
- そのまま続けてSTP・4P・3C・BMC分析を実施し、概要設計書の草稿を生成
- 担当者はレビュー・修正のみで済む状態にする

**必要な連携：**


| 要素        | 候補ツール                                     |
| --------- | ----------------------------------------- |
| 文字起こし     | Zoom / Google Meet の文字起こし機能、Notta、Whisper |
| AI分析・文章生成 | Claude API / ChatGPT API                  |
| 出力先       | Notion API（議事録DB・概要設計書）、Google Docs       |


**実装イメージ：**

```
MTG録音 → 文字起こし → Claude API（議事録整理＋分析） → Notion自動格納
```

## ② プレゼン資料の自動作成

**対象フェーズ**：フェーズ1（商談・ファーストヒアリング）

**現状の課題：**

- Canvaのテンプレートを毎回複製して、クライアント情報を手動で差し替えている
- 業界・ターゲット・課題に応じた内容のカスタマイズに時間がかかる

**自動化アイデア：**

- Notionに登録したクライアント情報（業界・受注内容など）を入力として使う
- AIがプレゼン内容（訴求ポイント・事例・提案構成）を生成
- Canva APIまたはGoogle Slides APIで資料に自動反映

**必要な連携：**


| 要素      | 候補ツール                         |
| ------- | ----------------------------- |
| データ取得   | Notion API                    |
| コンテンツ生成 | Claude API / ChatGPT API      |
| 資料反映    | Canva API / Google Slides API |
| 自動化フロー  | Make / Zapier / GAS           |


**実装イメージ：**

```
Notionに顧客情報登録 → Claude API（スライド内容生成） → Canva/Google Slides自動生成
```

## ③ 契約書作成の自動化

**対象フェーズ**：フェーズ2（受注後）

**現状の課題：**

- 毎回Google Docsのテンプレートを複製してクライアント名・金額・日付を手動で差し替えている
- 送付文もLINEに手動でコピペして送っている

**自動化アイデア：**

- Notionの案件レコード（クライアント名・受注金額・受注日）をトリガーに
- Google Docsのテンプレートを自動複製・必要箇所を自動入力してドライブに格納
- LINEへ契約書URLと定型送付文を自動送信

**必要な連携：**


| 要素       | 候補ツール                   |
| -------- | ----------------------- |
| トリガー     | Notion API（ステータス変更イベント） |
| ドキュメント生成 | Google Docs API / GAS   |
| ドライブ格納   | Google Drive API        |
| LINE送信   | LINE Messaging API      |
| 自動化フロー   | Make / Zapier / GAS     |


**実装イメージ：**

```
Notionステータスが「見込み→受注」に変更
→ Google Docsテンプレを自動複製・クライアント情報を自動入力
→ 顧客フォルダへ格納
→ LINEに契約書URL＋送付文を自動送信
```

## ④ 案件進行状況のNotion管理

**対象フェーズ**：全フェーズ

**現状の課題：**

- 各フェーズの完了を手動でNotionのステータスに反映している
- どの案件がどのフェーズにいるのかをNotionで一目で把握しにくい
- チェックリストの消化状況がNotionに反映されない

**自動化アイデア：**

- フェーズごとのチェックリストをNotionのデータベースで管理
- 各タスク完了時に自動でフェーズ進捗が更新される仕組みを構築
- ダッシュボードビューで「フェーズ別」「担当者別」「期限別」の進捗を可視化

**Notionで実現したい機能：**


| 機能           | 内容                             |
| ------------ | ------------------------------ |
| 案件DB         | クライアント名・フェーズ・担当者・期限・進捗ステータス    |
| フェーズ別チェックリスト | 各フェーズのタスクをサブページ or チェックボックスで管理 |
| ダッシュボード      | フェーズ別・担当者別・期限別のビューを作成          |
| 自動ステータス更新    | Notionのリレーション＋ロールアップで自動集計      |
| 期限アラート       | Notionの期限プロパティ＋SlackまたはLINE通知  |


**実装イメージ：**

```
案件DB（Notion）
├─ フェーズ1：商談・ヒアリング   [チェックリスト]
├─ フェーズ2：受注後           [チェックリスト]
├─ フェーズ3：キックオフMTG前   [チェックリスト]
├─ フェーズ4：概要設計         [チェックリスト]
├─ フェーズ5：詳細設計         [チェックリスト]
├─ フェーズ6：構築実装         [チェックリスト]
├─ フェーズ7：納品後           [チェックリスト]
└─ フェーズ8：運用             [チェックリスト]
```

## ⑤ 配信動線から配信内容ベースの自動生成

**対象フェーズ**：フェーズ8（運用）

**現状の課題：**

- 配信テーマを選んでから毎回ゼロベースでシナリオ・配信文を作成している
- テーマごとにターゲット・ベネフィット・CTA（行動喚起）を考える工数が大きい

**自動化アイデア：**

- 配信テーマ一覧・クライアントのペルソナ・過去の概要設計書を入力として与える
- AIが配信動線（見込み客の行動フロー）を設計し、各ステップの配信文のベースを生成
- 担当者はレビュー・微調整のみで配信文を完成させられる状態にする

**必要な連携：**


| 要素      | 候補ツール                                                   |
| ------- | ------------------------------------------------------- |
| 入力データ   | 配信テーマ一覧（Google Spreadsheet）、概要設計書（Google Docs / Notion） |
| コンテンツ生成 | Claude API / ChatGPT API                                |
| 出力先     | Lステップ（配信文として貼り付け）、Google Docs（草稿として保存）                  |


**実装イメージ：**

```
配信テーマ選択＋ペルソナ情報入力
→ Claude API（配信動線設計＋各ステップの配信文生成）
→ Google Docsに草稿として出力 → 担当者がレビュー・Lステップに登録
```

**シナリオ配信の生成ルール（組み込む）：**

- 通知欄は30文字前後
- 最初の2行で読者の興味を引く

## 定型作業の自動化（既存フローの効率化）

### Googleドライブ顧客フォルダ＋ファイル複製

**フェーズ**：1

**自動化内容：** Notionに新規案件を登録したタイミングで、Googleドライブに顧客フォルダを自動作成し、以下のファイルを自動複製・命名して格納する。


| ファイル           | 複製元     |
| -------------- | ------- |
| 議事録・アーカイブ管理シート | 原本スプシ   |
| 進行シート          | 本ドキュメント |
| ヒアリングシート       | 原本Docs  |
| 見積書            | 原本スプシ   |
| 月次運用プラン表       | 原本スプシ   |


**ツール候補**：Google Drive API ＋ GAS または Make / Zapier

---

### Notion進捗ステータス更新

**フェーズ**：2・7

**自動化内容：** トリガーとなる操作（請求書送付・入金確認・納品MTG完了など）に連動してNotionの進捗ステータスを自動更新する。


| トリガー          | ステータス変更          |
| ------------- | ---------------- |
| 請求書送付（Misoca） | 見込み → 受注確認中      |
| 入金確認          | 受注確認中 → 進行中（実績）  |
| 納品MTG完了       | 進行中（実績） → 完了（実績） |


---

### LINEノート定型文投稿

**フェーズ**：3・5

**自動化内容：** フェーズ移行時に定型のLINEノート内容をLINE Messaging APIで自動投稿し、アナウンス固定する。

---

### 議事録送付リマインダー

**フェーズ**：4〜6

**自動化内容：** MTG実施の記録（カレンダーまたはNotionへの登録）をトリガーに、24時間後に議事録未送付の場合はLINEまたはSlackでリマインダーを送信する。

---

### 納品後フォローアップ連絡

**フェーズ**：7

**自動化内容：** 納品完了日をNotionに記録したタイミングで、1週間後・1ヶ月後の日時にLINEメッセージを自動送信する。

---

### 毎週月曜の挨拶連絡

**フェーズ**：7・8

**自動化内容：** 運用中案件のクライアントへ、毎週月曜日の朝にLINE Messaging APIで定型挨拶を自動送信する。

---

### 月次レポート作成

**フェーズ**：8

**自動化内容：** Lsearch APIでKPIデータを取得し、Canva APIでレポートテンプレに自動反映して生成する。

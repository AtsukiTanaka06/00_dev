## フェーズ4：概要設計開始〜MTG準備

| 項目 | 内容 |
| --- | --- |
| **インプット** | キックオフMTG議事録、ヒアリングシート回答、業界・事業情報 |
| **アウトプット** | マーケティング分析資料（AI活用）、スケジュール進行表、プロジェクトシート（Miro）、概要設計書、シナリオ配信骨子、概要設計MTGアジェンダ |
| **利用サービス** | AI（ChatGPT等）、Google Spreadsheet、Miro、Google Docs |

### 全体ルール（フェーズ4以降共通）

- 議事録はどれだけ遅くとも **MTGから24時間以内** に送付する
- 制作物や共有物はフォルダ分けして **Googleドライブへ格納** する

### 4-1. リサーチ（AI活用）

以下の順でAIを使ってクライアント事業を分析する。

- [ ] **ゴール設定**：目的・目標を明確化
- [ ] **ペルソナ＆ベネフィット**：ペルソナ / 課題 / 解決策 / ベネフィット
- [ ] **STP分析**：セグメント / ターゲット / ポジション
- [ ] **4P分析**：商品サービス / 価格 / 販売場所や提供方法 / 販売促進
- [ ] **3C分析**：顧客 / 競合 / 自社
- [ ] **ビジネスモデルキャンバス**（9要素）
  - 顧客セグメント
  - 価値提案
  - チャネル
  - 顧客との関係
  - 収益の流れ
  - 主要リソース
  - 主要活動
  - 主要パートナー
  - コスト構造

### 4-2. 要件定義：プロジェクトシート作成

- [ ] [スケジュール進行表（原本）](https://docs.google.com/spreadsheets/d/16rsCLeAFYOEATYSD6Gpm68Wc9p_QMRopJzYcVJWVtLo) を複製して全体スケジュールを確定させる
- [ ] [プロジェクトシート（Miro原本）](https://miro.com/app/board/uXjVJMsJIcY=) を複製してシナリオの概要構成・リッチメニューの概要設計を作成

### 4-3. 概要設計書作成

- [ ] [概要設計書（原本）](https://docs.google.com/document/d/17F1A7cw5JWjPAKylsX_6YOzwWRn45xj3XvjF6vnPg98) を複製して全体概要設計書を作成（※原本のため必ず複製すること）
- [ ] [シナリオ配信骨子テンプレ](https://docs.google.com/document/d/19hgpZMXQ_z8v_MuZoEafIH0-ZeFeiRe8jeuxtya2HZ4) を参照してシナリオ配信骨子を作成
  - **ポイント**：通知欄は30文字前後、最初の2行にこだわること

### 4-4. 概要設計MTGアジェンダ作成

- [ ] [アジェンダ（原本）](https://docs.google.com/document/d/1p2VTBZUrcxRqOYC5jS3-2OKZJOqc6hkZ9NYyzWEXvY4) を複製して概要設計MTG用に編集

| タイミング | 内容 |
| --- | --- |
| **納品1週間後** | 状況確認の連絡を行う |
| **納品1ヶ月後** | KPIとの数字の差分を **必ず西田氏に報告**。運用契約の有無に関わらず運用状況・成果を確認 |
| **運用フェーズ中（毎週月曜）** | 挨拶の連絡を送る |
| **随時** | 定期配信の挙動チェックを怠らない |

## Notion 進捗ステータス遷移

```
見込み  →（入金確認後）→  進行中（実績）  →（納品後）→  完了（実績）
```

| ステータス | 変更タイミング | 担当 |
| --- | --- | --- |
| 見込み | 案件レコード作成時（初期値） | フェーズ1 |
| 進行中（実績） | 入金確認後 | フェーズ2 |
| 完了（実績） | 納品後 | フェーズ7 |

---

> 自動化候補については `02_自動化アイデアメモ.md` を参照。

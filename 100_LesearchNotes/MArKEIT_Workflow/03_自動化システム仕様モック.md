# 自動化システム仕様・モック

## なぜ n8n で全て作れるのか

### n8n とは
ワークフロー自動化ツール。「ノード（処理ブロック）」をつなげてフローを組む。
Make / Zapier に似ているが、**コードノードで複雑なロジックも書ける**のと、
**AIノード（Claude / OpenAI）が標準搭載**されている点が決定的な違い。

### 他ツールとの比較

| 機能 | n8n | Make | Zapier | GAS（個別） |
| --- | --- | --- | --- | --- |
| AI（Claude / GPT）との連携 | ✅ 標準搭載 | 🔧 HTTP経由 | 🔧 HTTP経由 | 🔧 自前実装 |
| 複雑なコードロジック | ✅ Codeノード（JS） | ❌ 式のみ | ❌ 式のみ | ✅ |
| フォームUIの内蔵 | ✅ Form Trigger | ❌ | ❌ | ❌ |
| Notion連携 | ✅ 標準搭載 | ✅ | ✅ | 🔧 HTTP経由 |
| Google Docs / Drive連携 | ✅ 標準搭載 | ✅ | ✅ | ✅ 標準 |
| LINE Messaging API | 🔧 HTTP経由 | 🔧 HTTP経由 | 🔧 HTTP経由 | 🔧 HTTP経由 |
| スケジュール実行（Cron） | ✅ | ✅ | ✅ | ✅ |
| セルフホスト（コスト管理） | ✅ 無料 | ❌ | ❌ | ✅（GCP無料枠） |
| 全ワークフローを一元管理 | ✅ | ✅ | ✅ | ❌ スクリプト分散 |

→ **n8n が唯一「全要件を一つの画面で管理できる」ツール**

### 必要な外部サービス（n8n から呼び出す）

n8n はあくまで「処理をつなぐ基盤」。以下のサービスのAPIキーを n8n に登録して使う。

| サービス | 用途 | 費用感 |
| --- | --- | --- |
| Claude API（Anthropic） | AI文章生成・分析 | 従量課金（$3〜15 / 100万トークン） |
| Notion API | 案件DB読み書き | 無料（Notionプランによる） |
| Google Workspace API | Docs / Drive / Slides操作 | 無料（GCPプロジェクト作成が必要） |
| LINE Messaging API | LINE自動送信 | 無料枠あり（月200通まで無料） |
| Misoca API | 請求書自動作成 | Misocaプランによる |
| n8n 本体 | ワークフロー実行基盤 | セルフホスト無料 / クラウド $20〜/月 |

## ワークフロー① 議事録〜リサーチのAI実施

**フェーズ**：4（概要設計開始）
**トリガー**：n8n フォーム（担当者が文字起こしテキストを貼り付けて送信）

### フロー図

```
[Form Trigger]
  入力項目：
  ・クライアント名（Notionの案件を選択）
  ・MTG文字起こしテキスト（貼り付け）
  ・MTG種別（キックオフ / 概要設計 / 詳細設計 / 納品）
        ↓
[Notion node]
  案件レコードからペルソナ・業界・受注内容を取得
        ↓
[Anthropic node]  ← 議事録整理プロンプト
  出力：構造化議事録
  ・日時・参加者・議題・決定事項・ネクストアクション
        ↓
[Anthropic node]  ← マーケティング分析プロンプト（キックオフMTGのみ）
  出力：
  ・ゴール設定 / ペルソナ＆ベネフィット / STP / 4P / 3C / BMC
        ↓
[Notion node]
  議事録ページを案件レコード配下に自動作成・格納
        ↓
[Google Docs node]
  概要設計書テンプレートを複製し、分析結果を自動入力
        ↓
[Google Drive node]
  顧客フォルダへ移動・命名
```

### フォームのイメージ

```
┌─────────────────────────────────────┐
│  議事録・リサーチ 自動生成フォーム          │
├─────────────────────────────────────┤
│  案件名   [ プルダウン（Notionから取得）]   │
│  MTG種別  [ ○キックオフ ○概要 ○詳細 ○納品] │
│  文字起こし [          テキストエリア      ] │
│                                      │
│            [ 送信して自動生成する ]         │
└─────────────────────────────────────┘
```

## ワークフロー② プレゼン資料の自動作成

**フェーズ**：1（商談前）
**トリガー**：Notion trigger（案件レコードが新規作成されたとき）

### フロー図

```
[Notion Trigger]
  条件：案件レコードが新規作成された
        ↓
[Notion node]
  案件情報取得：クライアント名・業界・受注内容・担当者
        ↓
[Anthropic node]  ← プレゼン構成生成プロンプト
  出力：
  ・スライドごとのタイトルと本文（業界に合わせた訴求・事例）
        ↓
[Google Drive node]
  プレゼンテンプレートを複製・クライアント名でリネーム
        ↓
[Google Slides node]
  プレースホルダーをAI生成テキストで置換
  （{{クライアント名}}・{{業界課題}}・{{提案内容}} など）
        ↓
[Google Drive node]
  顧客フォルダへ移動
        ↓
[Notion node]
  案件レコードへプレゼン資料URLを記録
```

### Googleスライドのプレースホルダー設計

| スライド | プレースホルダー |
| --- | --- |
| 表紙 | `{{クライアント名}}`, `{{日付}}` |
| 課題提起 | `{{業界課題}}`, `{{現状の問題点}}` |
| 解決策 | `{{Lステップ活用提案}}` |
| 事例 | `{{類似業界事例}}` |
| 費用・スケジュール | `{{受注内容}}`, `{{期限}}` |

## ワークフロー③ 契約書作成の自動化

**フェーズ**：2（受注後）
**トリガー**：Notion trigger（案件ステータスが「受注確定」に変更されたとき）

### フロー図

```
[Notion Trigger]
  条件：進捗ステータスが「受注確定」に変更
        ↓
[Notion node]
  案件情報取得：クライアント名・受注金額・受注日・担当者
        ↓
[Google Drive node]
  契約書テンプレートを複製・クライアント名でリネーム
        ↓
[Google Docs node]
  プレースホルダーを案件情報で置換
  （{{クライアント名}}・{{受注金額}}・{{契約日}} など）
        ↓
[Google Drive node]
  顧客フォルダへ移動
        ↓
[HTTP Request node]  ← LINE Messaging API
  クライアントのLINEへ送信：
  ・定型送付文テンプレート
  ・契約書ドキュメントURL（自動埋め込み）
        ↓
[Notion node]
  案件レコードへ契約書URL・送付日を記録
```

### 契約書のプレースホルダー設計

| 箇所 | プレースホルダー |
| --- | --- |
| 冒頭（甲の名称） | `{{クライアント名}}` |
| 契約日 | `{{契約日}}` |
| 受注内容 | `{{受注内容}}` |
| 受注金額 | `{{受注金額}}` |
| 担当者名 | `{{担当者名}}` |

## ワークフロー⑤ 配信動線から配信内容ベースの自動生成

**フェーズ**：8（運用）
**トリガー**：n8n フォーム（担当者が配信テーマと対象を入力して送信）

### フロー図

```
[Form Trigger]
  入力項目：
  ・案件名（Notionから選択）
  ・配信テーマ（テキスト入力 or プルダウン）
  ・配信ゴール（例：セミナー申込み・LINE登録・来店）
  ・配信通数（例：3通 / 5通 / 7通）
        ↓
[Notion node]
  概要設計書・ペルソナ情報を取得
        ↓
[Anthropic node]  ← 配信動線設計プロンプト
  出力：
  ・配信ステップごとのゴール・タイミング・役割
        ↓
[Anthropic node]  ← 各ステップの配信文生成プロンプト
  出力（各通ごとに）：
  ・通知欄テキスト（30文字以内）
  ・本文（冒頭2行で興味を引く構成）
  ・CTA（行動喚起ボタン文言）
        ↓
[Google Docs node]
  配信文草稿としてドキュメントに自動出力
  （担当者がレビュー後にLステップへ登録）
        ↓
[Notion node]
  案件レコードへ草稿URLを記録
```

### フォームのイメージ

```
┌─────────────────────────────────────┐
│  配信コンテンツ 自動生成フォーム            │
├─────────────────────────────────────┤
│  案件名   [ プルダウン（Notionから取得）]   │
│  配信テーマ [                        ]   │
│  配信ゴール [                        ]   │
│  配信通数  [ ○3通 ○5通 ○7通 ○カスタム ] │
│                                      │
│            [ 配信文を自動生成する ]          │
└─────────────────────────────────────┘
```

## ワークフロー：議事録送付リマインダー

**フェーズ**：4〜6
**トリガー**：Notion trigger（議事録ページが作成されたとき）

### フロー図

```
[Notion Trigger]
  条件：案件配下に議事録ページが作成された
        ↓
[Wait node]
  20時間待機（24時間ルールの猶予として）
        ↓
[Notion node]
  議事録ページの「送付済みフラグ」を確認
        ↓
[IF node]
  送付済み → 何もしない（ワークフロー終了）
  未送付  → 続行
        ↓
[HTTP Request node]  ← LINE Messaging API
  担当者LINEへリマインダー送信：
  「【リマインド】○○様の議事録が未送付です。送付後に送付済みフラグを更新してください。」
```

## ワークフロー：毎週月曜の挨拶連絡

**フェーズ**：7・8
**トリガー**：スケジュール（毎週月曜日 AM9:00）

### フロー図

```
[Schedule Trigger]
  設定：毎週月曜日 09:00
        ↓
[Notion node]
  進捗ステータスが「進行中（実績）」の案件一覧を取得
        ↓
[Loop node]  ← 案件ごとに繰り返す
        ↓
[HTTP Request node]  ← LINE Messaging API
  各案件の担当者LINEグループへ送信：
  「おはようございます！今週もよろしくお願いします。
   今週のネクストアクション：〇〇（Notionから取得）」
```

## 開発ロードマップ（推奨順）

n8n を使って段階的に構築していく想定。

```
Phase 1：基盤構築（1〜2週間）
├─ n8n のセットアップ（クラウド or サーバー）
├─ 各サービスのAPIキー取得・n8nへ登録
│   Notion / Google Workspace / Claude / LINE
└─ ④ 案件進行状況のNotion管理（DB設計・Notionオートメーション）

Phase 2：ファイル自動化（1〜2週間）
├─ Googleドライブ顧客フォルダ＋ファイル複製
└─ ③ 契約書作成の自動化

Phase 3：AI活用（2〜3週間）
├─ ① 議事録〜リサーチのAI実施
└─ ⑤ 配信動線から配信内容ベースの自動生成

Phase 4：通知・フォローアップ自動化（1週間）
├─ 議事録送付リマインダー
├─ 納品後フォローアップ連絡
└─ 毎週月曜の挨拶連絡

Phase 5：資料自動生成（2〜3週間）
└─ ② プレゼン資料の自動作成
```

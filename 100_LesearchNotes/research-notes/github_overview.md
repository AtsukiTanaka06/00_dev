# GitHub概要

GitHubはソフトウェア開発プロジェクトのためのホスティングサービスで、Gitバージョン管理システムを基盤としています。

## 主な概念

- **リポジトリ（Repository）**: プロジェクトのファイルや履歴を保存する場所。ローカルとリモートの両方が存在する。
- **コミット（Commit）**: ファイルの変更を記録する操作。メッセージを付けて履歴に残す。
- **ブランチ（Branch）**: 開発の分岐を作る仕組み。`main`や`master`が通常のメインブランチ。
- **マージ（Merge）**: ブランチでの変更を別のブランチに統合する。
- **プルリクエスト（Pull Request）**: 変更をレビューし、マージを依頼するためのGitHub独自の仕組み。
- **フォーク（Fork）**: 他のユーザーのリポジトリを自分のアカウントにコピーして独立して開発する。
- **クローン（Clone）**: リモートリポジトリをローカルに複製する。
- **プッシュ（Push）**: ローカルのコミットをリモートリポジトリに送信する。
- **プル（Pull）/フェッチ（Fetch）**: リモートの変更をローカルに取り込む。
- **Issues**: バグや機能リクエスト、タスクなどを追跡する機能。
- **README.md**: リポジトリのトップページに表示される説明ファイル。

## 基本的な使い方

1. **リポジトリの作成**:
   - GitHub上で新しいリポジトリを作成。
   - `git init`、`git remote add origin <URL>` を使ってローカルに関連付け。
2. **変更の記録**:
   - 変更したファイルを `git add <file>` でステージし、`git commit -m "メッセージ"` でコミット。
3. **ブランチ管理**:
   - `git branch <name>` で新規ブランチを作成し、`git checkout <name>` または `git switch <name>` で切り替える。
   - `git merge <branch>` でブランチを統合。
4. **リモート操作**:
   - `git push origin <branch>` でリモートに送信。
   - `git pull` や `git fetch` で最新の変更を取得。
5. **プルリクエスト**:
   - GitHubのウェブUIで新しいプルリクエストを開いて、変更のレビューとマージを依頼。
6. **コラボレーション**:
   - 他の開発者とブランチやプルリクエストを通じて協働。
   - Issuesでバグ報告やタスク管理。

## マークダウンの活用

- GitHubでは `.md` ファイルを用いてドキュメントを記述。
- `README.md` などに記載すればリポジトリページに自動表示される。

## その他の機能

- **GitHub Actions**: CI/CDの自動化ワークフロー。
- **Projects**: カンバンボード形式でタスク管理。
- **Wiki**: プロジェクト用のドキュメントスペース。
- **Release**: ソフトウェアのバージョン公開。
